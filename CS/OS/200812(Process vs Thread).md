### Q1. 프로세스와 쓰레드의 차이는?

```
 A1. 프로세스는 현재 실행중인 프로그램을 말하고 디스크로부터 적재되어 CPU의 할당을 받을 수 있다.

    즉, 운영체제로부터 주소 공간, 파일, 메모리 등을 할당받으며 이것들을 총칭하는 개념이다.

    프로세스 = 프로세스 스택 + 데이터 섹션 + 힙

        프로세스 스택 : 함수의 매개변수, 복귀 주소, 로컬 변수 등 전달
        데이터 섹션 : 전역 변수
        힙 : 실행 중 동적으로 할당되는 메모리

 - 그래서 쓰레드와 차이는 아직 좀 더 공부하고 새로 적자.
```

### Q2. 프로그램과 프로세스의 차이는?

```
  A2. 프로그램이란 보조 기억장치(하드디스크, SSD)에 존재하며 실행되기를 기다리는 명령어와 데이터의의 묶음일 뿐이다. 즉, 이 프로그램의 명령어와 전적 데이터가 메모리에 적재되면 생명이 있는 프로세스가 된다.

      프로세스란 실행 중인 프로그램이라 말할 수 있다.
```

### Q3. 프로세스의 구성은 어떻게?

```
   A3. 프로세스란 프로그램을 실행시키기 위한 여러 도구 모음이라 생각합니다. 이 도구들은 PCB(Process Control Block)에 저장되고, 안에는 PID(프로세스 고유 ID), 프로세스의 상태를 나타내는 값, PC(Program Counter, 다음 실행 될 주소 저장), 스케줄링값, 프로세스 별 접근권한, 자기 자신의 위치를 알려주는 포인터값 등등 저장되어 있다.

   - 핵심 단어 위주로 다시 정리하자.
```
